3.4 #mixins.layout Layout

3.4.1 #mixins.layout.b .b

  • @sides: <side>* | x | y | h | v
    Sides of the box to apply border
  • @values: <length>*
    Border values

Sets up borders.

This is the alias of .box(border; @sides; @values)

3.4.2 #mixins.layout.b-property .b (+Property)

  • @prop: width | style | color
    The border property to set up
  • @sides: <side>* | -<side> | x | y | h | v
    Sides of the box to set up
  • @values: <border-@{prop}>*
    Values to apply for border property

Sets up borders' width, style, color.

This is the alias of .box(border; @sides; @values; @prop)

3.4.3 #mixins.layout.box .box

  • @area: margin | padding | border
    The area to set up
  • @sides: <side>* | -<side> | x | y | h | v
    Sides of the box to set up
    • x = h = left right
    • y = v = top bottom
    • -<side> means "All sides except it."
      Ex. -top = right bottom left
  • @values: <@{area}>* | <border-@{prop}>*
    Values for box model properties
  • @prop: "" | width | style | color [""]
    Border property (Only if @area = border)

Sets up box model properties (padding, border, margin).

This mixin is complex and thus you type a lot. So we recommend for using following mixins instead. These mixins invoke .box internally but need less parameters.

  • .m (For margins)
  • .p (For paddings)
  • .b (For borders)

3.8.5 #mixins.util.for-list .for (List)

  • @values <any[*]>
    A list of values
  • @ruleset <ruleset>
    A ruleset to invoke

Performs iteration over a list.

You must declare the mixin: .each(<integer>; <any>) inside @ruleset.
The 1st parameter is the iteration count starts from 1.
The 2nd parameter is each item of the @values.
